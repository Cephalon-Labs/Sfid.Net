# Sfid.Net 1.0.1

Patch release focused on packaging and release automation.

## Highlights

- Derives package versions directly from `AssemblyVersion` for both `Sfid.Net` and `Sfid.EntityFramework`.
- Publishes packages to GitHub Packages automatically when a matching GitHub Release is published.
- Simplifies local packaging and publishing so `pwsh ./eng/Pack.ps1` and `pwsh ./eng/Publish-NuGet.ps1` work without a manual version argument.
- Keeps the CI and release pipeline aligned by validating that publishable projects share the same assembly version.
- Includes minor cleanup in the core and EF integration projects.

## Packages

- `Sfid.Net`
- `Sfid.EntityFramework`

## Validation

- `dotnet test Sfid.Net.slnx`
- `pwsh ./eng/Pack.ps1`

## Notes

Create the GitHub Release with tag `1.0.1` or `v1.0.1` so the release workflow can validate the version and publish the generated `.nupkg` files to GitHub Packages automatically.
